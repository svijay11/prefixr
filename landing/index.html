Install free

The cache scheduler for back-to-back agent turns

Prefixr intercepts your LLM API calls locally and decides every turn whether to preserve your prefix cache or summarize now — saving you money over the next five.

install — prefixr
$ curl -fsSL https://raw.githubusercontent.com/svijay11/prefixr/main/install.sh | bash
# or install with pip
$ python3 -m pip install --user "git+https://github.com/svijay11/prefixr.git"
→ Prefixr installed.
$ prefixr init
$ prefixr run

Click any command to copy · Requires Python 3.9+ and git

Prefixr helps you before, during, and after every API call

Cache optimization that runs on your machine, not in the cloud.

Start every turn with a cost decision

Prefixr evaluates cost_preserve(N) vs cost_summarize(N) using real provider cache pricing. If preserving your prefix cache is cheaper over the next five turns, it leaves context alone. If not, it compresses the volatile tail.

# optimizer reasoning
action: "preserve"
hit_rate: 0.873
horizon_turns: 5
expected_savings: $0.00189

Align context without breaking cache

Before your payload leaves your machine, Prefixr tries anchor splitting, padding injection, and timestamp scrubbing — restructuring messages so stable blocks stay frozen and volatile noise doesn't bust your prefix.

# strategies evaluated in order
1. anchor_split — freeze system + docs
2. pad — align to 1024/2048 checkpoint
3. scrub — replace UUIDs, timestamps
4. summarize — compress via haiku/mini

Track every cache hit in real time

After each response, Prefixr reads cache token data from your provider and writes ground-truth stats to your local SQLite ledger. The dashboard updates live via WebSocket.

# ledger entry
tokens_input: 18420
tokens_cached: 16080
cache_hit_rate: 87.3%
cost_saved_usd: $0.00352
